# Test Records

Curated test result snapshots for key milestones. Not every run — only meaningful ones.

## When to save a record
- Before a customer delivery
- After a major bug fix
- When establishing a new baseline

## How to save
Copy the report from `local/reports/` and rename it clearly:
`YYYY-MM-DD_<context>.md`
Example: `2026-04-05_pre-first-customer.md`

## What lives here vs local/
- `test-records/` — committed, shared with team, permanent record
- `local/reports/` — all generated output, stays on your machine
